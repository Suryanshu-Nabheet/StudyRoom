# Contributing to AI WebRTC Study Rooms

Thank you for your interest in contributing! This document provides guidelines and instructions for contributing to this project.

## Getting Started

1. Fork the repository
2. Clone your fork: `git clone https://github.com/your-username/ai-webrtc-study-rooms.git`
3. Create a branch: `git checkout -b feature/your-feature-name`
4. Install dependencies: `npm install`
5. Set up environment variables (see README.md)

## Development Workflow

1. Make your changes
2. Test thoroughly
3. Ensure code follows the existing style
4. Commit with clear messages
5. Push to your fork
6. Open a Pull Request

## Code Style

- Use TypeScript for all new code
- Follow existing code formatting (Prettier/ESLint)
- Write clear, self-documenting code
- Add comments for complex logic
- Keep components focused and modular

## Pull Request Process

1. Update README.md if needed
2. Ensure all tests pass
3. Request review from maintainers
4. Address any feedback
5. Wait for approval before merging

## Reporting Issues

When reporting issues, please include:
- Clear description of the problem
- Steps to reproduce
- Expected vs actual behavior
- Browser/OS information
- Relevant console errors

## Feature Requests

For feature requests:
- Check if it's already requested
- Provide clear use case
- Explain the expected behavior
- Consider implementation complexity

Thank you for contributing!
